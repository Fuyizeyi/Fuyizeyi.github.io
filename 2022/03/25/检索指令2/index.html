检索指令#2--locate\which\whereis\grep

发表于2022-03-25|更新于2022-03-26
|字数总计:694|阅读时长:2分钟|阅读量:

一、 locate

（是在一个文件目录的资料库寻找，并不是像find在文件夹下直接进行检索，更快）

  • locate 的检索用资料库位置可以使用 locate locate来检索自身位置，但是可能因为新创建的文件，还没更新locate的资料库，可能会出现查找不到，需要使用指令updatedb更新资料库，而find是实时查找，并不需要资料库。
  • tmp文件夹为临时文件夹，不会录入文件库中，使用了locate指令无法查询

  • 找不到 locate命令的话
    “locate 未找到此命令的”解决方法 输入 yum install mlocate 安装 然后updatedb更新库即可
  • 基础信息
    命令名称：locate
    命令所在路径：/usr/bin/locate
    执行权限：所有用户
    语法： 功能描述：在文件资料库中查找文件

二、which

1、基础信息

命令名称：which
命令所在路径：/usr/bin/which
执行权限：所有用户
语法：

功能描述：搜索命令所在目录及别名信息（实际传入参数）

2、基础使用

在查询指令的时候，他下面会显示出这条指令的别名和所在目录，此处以rm为例，rm的别名是“rm -i”，他所在的目录为/usr/bin/rm，我们在使用rm这条指令的时候，实际上用的是“rm -i”这个，rm本身是不存在询问是否删除的过程的，而 -i 正是这个询问的来源

三、whereis

命令名称：whereis
命令所在路径：/usr/bin/whereis
执行权限：所有用户
语法：

功能描述：搜索命令所在目录及帮助文档路径

2、基础使用

  • 查找 ls 的指令所在与相关帮助文档的文件夹

1、基础信息

命令名称：grep
命令所在路径：/bin/grep
执行权限：所有用户
语法：

功能描述：在文件中搜寻字串匹配的行并输出（找某个关键词所在的行）

2、使用
